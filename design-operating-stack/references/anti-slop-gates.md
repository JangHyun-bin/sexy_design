# Anti-Slop Gates

Run these before shipping study, audit, or generation output.

## Hard Fails

1. Generic centered SaaS hero by default:
   headline, paragraph, two buttons, three feature cards.
2. Purple-blue decorative gradient blobs without product reason.
3. Rounded cards used as filler.
4. Fake browser chrome, fake phone chrome, or fake OS title bars.
5. Fabricated metrics, testimonials, logos, awards, or user counts.
6. Hover scale as the primary interaction.
7. Mobile as stacked desktop sections.
8. Decorative charts that encode no real state or value.
9. Controls without state design.
10. More than one dominant gesture in the first viewport.
11. Accent color used without a footprint.
12. Product screenshot used only as decoration.

## Soft Fails

- Overly balanced 50/50 split when the reference language wants asymmetry.
- Too many equally loud text sizes.
- Border radius applied uniformly across unrelated surfaces.
- Shadows used to fake depth instead of material or state.
- Logos displayed as proof without context.
- Lottie-like motion that does not express state.
- Large text that says nothing specific.

## Mobile-Specific Gates

- Primary action is outside comfortable thumb reach without a reason.
- Tap targets below 44 by 44 px.
- Bottom nav or action bar lacks selected, pressed, and disabled states.
- Number picker, card stack, or control has no intermediate state.
- Modal or sheet ignores safe areas.
- Input focus state is visually weaker than default state.
- Loading state blocks the screen without feedback.

## Web-Specific Gates

- First viewport has no object, number, wordmark, product, or material scene.
- Logo wall appears before the page earns trust.
- Dashboard surface is decorative and has no operational logic.
- Hero visual is a generic app screenshot in a rounded card.
- Footer defaults to Product, Company, Resources, Legal without purpose.

## Gate Response

If a gate fails, do not rationalize it. Revise the design or clearly report the
failure in audit mode.
