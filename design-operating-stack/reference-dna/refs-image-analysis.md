## Cluster 1: Dark Technical Editorial

Closest source signal: anonymized cluster A.

```text
genre: Dark Technical Editorial
paper_band: dark
paper_hue: neutral black
accent_hue: hot orange
accent_footprint: pin
type_display_role: editorial serif
type_body_role: muted neutral sans
type_label_role: mono or uppercase technical
dominant_gesture: dark operational dashboard
object_type: dashboard surface
surface_treatment: dotted grid, halftone, hairline frame
web_pattern: operational-dashboard-surface
mobile_pattern: transaction-dashboard when adapted to phone
interaction_stance: still and precise, stateful dashboard
```

Transferable rules:

- Make the dashboard the object.
- Use orange only as status heat.
- Let hairlines, grid, and mono labels create precision.
- Keep motion small and operational.

Do not copy:

- Original brand, logo, exact copy, exact dashboard arrangement.


Transferable rules:

- Amounts and times should dominate when they are the product.
- Bottom actions and pressed states are part of the visual identity.
- One bold accent can own the screen if it has a control role.

Do not copy:

- Exact app layouts, financial brand marks, card art, account values, device
  mockup composition.

## Cross-Set Conclusions

- The set values object-first composition over template sections.
- The strongest references use one dominant gesture per viewport.
- Accent color is either tiny and hot or large and conceptually justified.
- Mobile must be treated as operated state: time, money, cards, controls,
  transactions, and bottom actions.
- Generic SaaS polish is weaker than material, state, and object authority.
