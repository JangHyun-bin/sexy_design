# Operational Dashboard Surface

## Use When

Use when a product's value is best shown through state, work, metrics, tasks,
charts, transactions, or monitoring.

## Anatomy

- dashboard shell or surface
- real or labeled placeholder metrics
- chart or task table
- status indicators
- compact labels
- surrounding proof or explanation

## Layout Rules

- The dashboard is an object, not a decoration.
- Use real values or clearly marked placeholders.
- Status, chart, and task areas should have believable relationships.
- Keep frames minimal and avoid fake browser chrome.

## Typography Rules

- Numbers use tabular figures.
- Labels can be mono or uppercase.
- Body copy outside the surface should explain the work, not repeat labels.

## Surface Rules

- Dark control-room and industrial gray surfaces work well.
- Hairlines, dotted grids, and small hot accents can show precision.
- Avoid soft generic cards.

## Responsive Behavior

- On mobile, simplify dashboard into primary metric, two to four tiles, and one
  list or chart.
- Do not shrink dense tables into unreadability.

## Anti-Slop Checks

- Fails if charts encode nothing.
- Fails if every metric is invented.
- Fails if browser chrome is hand-drawn around the surface.
