# Object-First Hero

## Use When

Use when a product, device, artifact, dashboard, chart, or physical object should
carry the first impression.

## Anatomy

- minimal navigation
- dominant object
- short annotation copy
- one primary action or no action
- optional spec labels
- continuation section that explains the object

## Layout Rules

- The object must occupy the strongest visual position.
- Copy annotates the object; it does not compete with it.
- Use asymmetry when the object has direction or weight.
- Leave enough negative space for authority.

## Typography Rules

- Display type can be smaller than the object.
- Labels can be mono or uppercase.
- Body copy should be short.

## Surface Rules

- Surface should support the object's material: industrial gray, dark control
  room, warm paper, or cinematic field.
- Avoid decorative backgrounds unrelated to the object.

## Responsive Behavior

- On mobile, object remains first or moves directly after a short title.
- Avoid tiny object thumbnails.
- If object detail is important, allow horizontal detail views or stacked crops.

## Anti-Slop Checks

- Fails if the object is a generic app screenshot in a rounded card.
- Fails if three feature cards become the real focus.
- Fails if copy explains what the object should show.
